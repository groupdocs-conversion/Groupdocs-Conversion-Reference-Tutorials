# تحويل MHTML إلى PSD باستخدام .NET باستخدام GroupDocs.Conversion

## مقدمة

في عصرنا الرقمي، تُعدّ إدارة المستندات بكفاءة أمرًا بالغ الأهمية. سواءً كنت تُحضّر مواد تسويقية أو تُؤرشف مستندات، فإن تحويل الملفات بين الصيغ المختلفة غالبًا ما يكون ضروريًا. سيُرشدك هذا الدليل إلى كيفية استخدام **GroupDocs.Conversion لـ .NET** لتحويل ملفات MHTML إلى صيغة PSD، مع الحفاظ على الدقة البصرية وتمكين المزيد من التحرير.

يتناول هذا البرنامج التعليمي تحدي تحويل محتوى MHTML على الويب (دمج HTML مع موارد مثل الصور) إلى مستند Photoshop احترافي (PSD). من خلال الاستفادة **GroupDocs.Conversion**يمكنك أتمتة هذه العملية بكفاءة داخل تطبيقات .NET الخاصة بك.

### ما سوف تتعلمه:
- إعداد GroupDocs.Conversion لـ .NET
- تحويل ملفات MHTML إلى صيغة PSD باستخدام C#
- تحسين أداء التحويل
- استكشاف الأخطاء وإصلاحها أثناء عملية التحويل

قبل أن نبدأ، دعونا نراجع المتطلبات الأساسية للتأكد من استعدادك.

## المتطلبات الأساسية

### المكتبات والتبعيات المطلوبة
لتنفيذ هذه الوظيفة، تأكد من:
- **GroupDocs.Conversion لـ .NET**:تم تثبيت الإصدار 25.3.0 أو الإصدار الأحدث.
- بيئة .NET مدعومة (على سبيل المثال، .NET Core 3.1+ أو .NET Framework 4.6.1+).

### إعداد البيئة
تأكد من أن بيئة التطوير لديك تتضمن Visual Studio أو أي بيئة تطوير متكاملة متوافقة تدعم C#. يجب أن يكون لديك أيضًا إمكانية الوصول إلى المجلدات لتخزين ملفات MHTML وحفظ ملفات PSD المُحوّلة.

### متطلبات المعرفة
- فهم أساسي لبرمجة C#.
- - المعرفة بكيفية التعامل مع الملفات في تطبيقات .NET.
- ستكون بعض المعرفة بمفاهيم تحويل المستندات مفيدة.

## إعداد GroupDocs.Conversion لـ .NET
للبدء، قم بتثبيت **GroupDocs.Conversion** المكتبة عبر وحدة تحكم مدير الحزم NuGet أو .NET CLI:

```plaintext
Install-Package GroupDocs.Conversion -Version 25.3.0
```

### .NET CLI
```bash
dotnet add package GroupDocs.Conversion --version 25.3.0
```

## دليل التنفيذ
يرشدك هذا القسم خلال عملية تحويل تنسيق MHTML إلى تنسيق PSD.

### تحميل وتحويل MHTML إلى PSD
**ملخص**:تتيح لك هذه الميزة تحميل ملف MHTML وتحويله إلى تنسيق PSD، وهو أمر مفيد لمصممي الرسوميات الذين يحتاجون إلى ملفات صور عالية الجودة لتحريرها في Adobe Photoshop.

#### الخطوة 1: تحديد الدلائل
إعداد أدلة الإدخال والإخراج لقراءة ملفات MHTML المصدر وحفظ ملفات PSD المحولة.

```csharp
string documentDirectory = "YOUR_DOCUMENT_DIRECTORY";
string outputDirectory = "YOUR_OUTPUT_DIRECTORY";
```

#### الخطوة 2: تهيئة المحول
إنشاء مثيل لـ `Converter` الفئة، وتوجيهها إلى ملف MHTML. هنا تبدأ GroupDocs.Conversion عملية التحويل.

```csharp
using (Converter converter = new Converter(Path.Combine(documentDirectory, "sample.mhtml")))
{
    // منطق التحويل يذهب هنا
}
```

#### الخطوة 3: تعيين خيارات التحويل
حدد تنسيق الإخراج المطلوب باستخدام `ImageConvertOptions`، تعيينه إلى PSD.

```csharp
ImageConvertOptions options = new ImageConvertOptions { Format = GroupDocs.Conversion.FileTypes.ImageFileType.Psd };
```

#### الخطوة 4: تحديد دالة تدفق الإخراج
إنشاء وظيفة تقوم بإنشاء تدفقات لكل صفحة مستند، مع التأكد من حفظ الملفات بشكل صحيح.

```csharp
string outputFileTemplate = Path.Combine(outputDirectory, "converted-page-{0}.psd");
Func<SavePageContext, Stream> getPageStream = savePageContext => new FileStream(string.Format(outputFileTemplate, savePageContext.Page), FileMode.Create);
```

#### الخطوة 5: تنفيذ التحويل
تنفيذ التحويل عن طريق الاتصال `converter.Convert`، تمرير وظيفة التدفق والخيارات الخاصة بك.

```csharp
converter.Convert(getPageStream, options);
```

### نصائح استكشاف الأخطاء وإصلاحها
- **تأكد من مسارات الملفات الصحيحة**:تحقق جيدًا من مسارات الدليل لتجنب أخطاء عدم العثور على الملف.
- **إدارة الذاكرة**: يستخدم `using` عبارات لإدارة التدفقات لمنع تسرب الذاكرة.
- **توافق الإصدار**:تأكد من استخدام إصدار متوافق من GroupDocs.Conversion مع بيئة .NET الخاصة بك.
