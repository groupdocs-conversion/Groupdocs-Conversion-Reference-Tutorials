# Как конвертировать защищённые паролем документы Word в Excel с помощью GroupDocs.Conversion для Java

Конвертация защищённых паролем документов Word в таблицы Excel — распространённая необходимость, когда требуется проанализировать или поделиться данными в табличном виде. В этом руководстве вы узнаете **как конвертировать защищённые паролем документы Word** в Excel с использованием GroupDocs.Conversion API для Java, шаг за шагом.

## Быстрые ответы
- **Могу ли я конвертировать защищённый файл Word без пароля?** Нет — необходимо предоставить правильный пароль через параметры загрузки.  
- **Какие форматы вывода поддерживаются для электронных таблиц?** Доступны XLS, XLSX, CSV и ODS.  
- **Нужна ли лицензия для использования в продакшене?** Да, требуется действующая лицензия GroupDocs для не‑тестовых развертываний.  
- **Достаточен ли Java 8?** Поддерживается Java 8+; более новые версии также работают.  
- **Можно ли конвертировать только определённые страницы?** Конечно — используйте `setPageNumber` и `setPagesCount` в параметрах конвертации.

## Что означает «convert password protected word»?
Эта фраза относится к процессу открытия документа Word, защищённого паролем, и последующего преобразования его содержимого в другой тип файла — в данном случае в электронную таблицу Excel. Это полезно для извлечения данных, составления отчётов и автоматизации рабочих процессов.

## Как загрузить защищённый паролем документ в Java

Для загрузки документа, защищённого паролем, требуется объект **load options**, содержащий пароль. Этот шаг разблокирует файл, позволяя конвертеру прочитать его содержимое.

```java
import com.groupdocs.conversion.options.load.WordProcessingLoadOptions;

WordProcessingLoadOptions loadOptions = new WordProcessingLoadOptions();
loadOptions.setPassword("12345"); // Replace with your document's password.
```

*Объяснение:* `WordProcessingLoadOptions` специально предназначен для файлов Word. Вызовом `setPassword` вы предоставляете API разрешение открыть защищённый документ.

## Конвертация Word в таблицу в Java — расширенные параметры

После загрузки документа вы можете задать, как должна происходить конвертация — выбор страниц, формата и визуального масштабирования.

```java
import com.groupdocs.conversion.Converter;
import com.groupdocs.conversion.filetypes.SpreadsheetFileType;
import com.groupdocs.conversion.options.convert.SpreadsheetConvertOptions;

SpreadsheetConvertOptions options = new SpreadsheetConvertOptions();
options.setPageNumber(2); // Start from page 2.
options.setPagesCount(1); // Convert only one page.
options.setFormat(SpreadsheetFileType.Xls); // Output format as XLS.
options.setZoom(150); // Set zoom level for conversion (scale factor).
```

*Объяснение:*  
- `setPageNumber` и `setPagesCount` позволяют указать конкретный диапазон страниц, что удобно, когда нужен только фрагмент документа.  
- `setFormat` выбирает контейнер таблицы (в данном случае XLS).  
- `setZoom` регулирует масштаб рендеринга, полезно для сохранения точности макета.

## Выполнение конвертации

Когда параметры загрузки и конвертации готовы, вызовите метод `convert`. API выполнит всю сложную работу в фоновом режиме.

```java
String convertedFile = Constants.getConvertedPath("ConvertToSpreadsheetWithAdvancedOptions.xls");
Converter converter = new Converter(Constants.SAMPLE_DOCX_WITH_PASSWORD, () -> loadOptions);
converter.convert(convertedFile, options);
```

*Объяснение:* Конструктор `Converter` получает путь к исходному файлу и лямбда‑выражение, предоставляющее ранее определённые `loadOptions`. Вызов `convert` записывает файл Excel в место, возвращаемое `Constants.getConvertedPath`.

## Распространённые проблемы и решения

| Симптом | Возможная причина | Решение |
|---------|-------------------|--------|
| **«Invalid password» исключение** | Неправильная строка пароля или проблема с кодировкой | Проверьте пароль, убедитесь, что нет лишних пробелов, и используйте точный регистр. |
| **Отсутствуют страницы в результате** | Неправильные значения `setPageNumber`/`setPagesCount` | Дважды проверьте номера страниц в исходном файле Word; помните, что нумерация начинается с 1. |
| **Ошибка Out‑of‑memory при больших файлах** | Загрузка всего документа в память | Обрабатывайте большие файлы по частям или увеличьте размер кучи JVM (`-Xmx`). |
| **Ошибка неподдерживаемого формата** | Использование более старой версии GroupDocs | Обновите до последней библиотеки (например, 25.2). |
