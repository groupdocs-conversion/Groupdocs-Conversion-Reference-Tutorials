## Wat is PDF‑naar‑JPG‑conversie in Java?
Een PDF‑document omzetten naar een reeks JPG‑afbeeldingen in Java betekent dat elke pagina (of geselecteerde pagina's) wordt geëxtraheerd en gerasterd tot bitmap‑afbeeldingen. Dit is handig voor het maken van miniaturen, voorbeeldafbeeldingen voor web‑viewers, of het voorbereiden van inhoud voor platforms die alleen afbeeldingsformaten accepteren.

## Waarom GroupDocs.Conversion voor Java gebruiken?
GroupDocs.Conversion abstraheert de low‑level renderlogica, zodat je je kunt concentreren op de applicatiestroom. Het ondersteunt:

* **Meerdere bronformaten** – PDF’s, DOCX, XLSX, PPTX, HTML en meer.  
* **Uitvoer van hoge kwaliteit** – Instelbare DPI, kleurdiepte en compressie‑instellingen.  
* **Stream‑vriendelijke API’s** – Werk direct met `InputStream`/`OutputStream` om schijf‑I/O te vermijden.  
* **Cross‑platform betrouwbaarheid** – Werkt in elke JVM‑compatibele omgeving.

## Veelgestelde vragen

**V: Kan ik alleen geselecteerde pagina's van een PDF naar JPG converteren?**  
A: Ja. De conversie‑API laat je een paginabereik of een array van paginanummers opgeven.

**V: Hoe regel ik de beeldkwaliteit van de JPG‑output?**  
A: Pas de `jpegQuality`‑instelling (0‑100) aan in het `JpgConvertOptions`‑object.

**V: Is het mogelijk om wachtwoord‑beveiligde PDF’s te converteren?**  
A: Absoluut. Geef het wachtwoord op bij het laden van de `ConversionConfig`.

**V: Wat is de beste DPI voor web‑klare miniaturen?**  
A: 72–96 DPI biedt een goede balans tussen kwaliteit en bestandsgrootte.

**V: Moet ik streams handmatig sluiten?**  
A: De bibliotheek sluit streams automatisch wanneer de conversie voltooid is, maar het is goede praktijk om aangepaste streams te sluiten in een `try‑with‑resources`‑blok.
