## कार्यान्वयन मार्गदर्शिका

यह अनुभाग .NET के लिए GroupDocs.Conversion का उपयोग करके DCM को PSD में परिवर्तित करने में आपकी सहायता करता है।

### रूपांतरण प्रक्रिया का अवलोकन

इसका लक्ष्य DICOM फ़ाइल को फ़ोटोशॉप-संगत प्रारूप में परिवर्तित करना है, जिससे ग्राफ़िक डिज़ाइन सॉफ़्टवेयर में हेरफेर करना आसान हो जाएगा।

#### चरण 1: आउटपुट निर्देशिका और टेम्पलेट सेट करें
परिभाषित करें कि परिवर्तित फ़ाइलें कहाँ संग्रहीत की जाएँगी और उनका नाम क्या होगा:

```csharp
string outputFolder = @"YOUR_OUTPUT_DIRECTORY";
string outputFileTemplate = Path.Combine(outputFolder, "converted-page-{0}.psd");
```

`outputFileTemplate` प्लेसहोल्डर का उपयोग करता है `{0}` यदि आपकी DCM फ़ाइल में एकाधिक पृष्ठ हैं, तो पृष्ठ संख्या के लिए क्लिक करें।

#### चरण 2: स्ट्रीम फ़ंक्शन परिभाषित करें
प्रत्येक रूपांतरित पृष्ठ के लिए आउटपुट स्ट्रीम को संभालने के लिए एक फ़ंक्शन बनाएँ:

```csharp
Func<SavePageContext, Stream> getPageStream = savePageContext =>
    new FileStream(string.Format(outputFileTemplate, savePageContext.Page), FileMode.Create);
```

यह फ़ंक्शन PSD फ़ाइलें लिखने के लिए एक नई फ़ाइल स्ट्रीम बनाता है।

#### चरण 3: स्रोत DCM फ़ाइल लोड करें और रूपांतरण विकल्प सेट करें
अपनी स्रोत DCM फ़ाइल लोड करें और रूपांतरण विकल्प कॉन्फ़िगर करें:

```csharp
using (Converter converter = new Converter("YOUR_DOCUMENT_DIRECTORY/SAMPLE_DCM"))
{
    ImageConvertOptions options = new ImageConvertOptions { Format = ImageFileType.Psd };
    
    // PSD प्रारूप में रूपांतरण निष्पादित करें
    converter.Convert(getPageStream, options);
}
```

`ImageConvertOptions` PSD आउटपुट के लिए कॉन्फ़िगर किया गया है। `converter.Convert()` विधि प्रत्येक पृष्ठ को संसाधित करती है और उसे एक अलग PSD फ़ाइल के रूप में लिखती है।

#### समस्या निवारण युक्तियों
- सुनिश्चित करें कि आपका DCM फ़ाइल पथ सही है.
- आउटपुट निर्देशिका पर अनुमतियों की जाँच करें.
- सत्यापित करें कि आपने GroupDocs.Conversion ठीक से स्थापित किया है।
