## Proč zvolit GroupDocs.Conversion pro Javu?

* **Široká podpora formátů** – Zpracovává Word, Excel, CSV, ZIP, e‑mail a mnoho dalších, což z něj činí jediné řešení pro všechny vaše potřeby konverze.  
* **Vysoká věrnost** – Zachovává rozvržení, fonty a složité prvky (tabulky, grafy, anotace), takže výstupní PDF vypadá přesně jako zdroj.  
* **Orientace na výkon** – Zjednodušená API umožňuje konvertovat velké dávky nebo jednotlivé soubory s minimální spotřebou paměti – ideální pro cloudové služby.  
* **Bezpečnost připravená** – Vestavěné možnosti pro práci s PDF chráněnými heslem, odstraňování vložených souborů a bezpečné získávání metadat.

## Rychlý start

1. **Přidejte Maven závislost** (nebo ekvivalent pro Gradle) pro GroupDocs.Conversion.  
2. **Vytvořte instanci `Converter`** s vaším licenčním klíčem.  
3. **Zvolte vstupní formát** (např. `.docx`, `.xlsx`, `.csv`).  
4. **Nakonfigurujte `PdfConvertOptions`** – nastavte velikost stránky, náhradu fontů nebo zpracování hesla.  
5. **Zavolejte `convert`** a zapište výstupní stream do souboru nebo odpovědi.

> **Pro tip:** Když potřebujete **convert excel to pdf java** s vlastními fonty, nastavte `PdfConvertOptions.setFontSubstitution(true)` a poskytněte `FontMapper`, aby byla zajištěna vizuální konzistence.
