## Dlaczego warto wybrać GroupDocs.Conversion dla Javy?

* **Szerokie wsparcie formatów** – Obsługuje Word, Excel, CSV, ZIP, e‑maile i wiele innych, będąc kompleksowym rozwiązaniem dla wszystkich potrzeb konwersji.  
* **Wysoka wierność** – Zachowuje układ, czcionki i złożone elementy (tabele, wykresy, adnotacje), dzięki czemu wygenerowany PDF wygląda dokładnie tak jak źródło.  
* **Skoncentrowane na wydajności** – Uproszczone API umożliwiają konwersję dużych partii lub pojedynczych plików przy minimalnym zużyciu pamięci – idealne dla usług w chmurze.  
* **Gotowe do zabezpieczeń** – Wbudowane opcje obsługi PDF‑ów chronionych hasłem, usuwania osadzonych plików i bezpiecznego wyodrębniania metadanych.

## Typowe przypadki użycia

| Scenariusz | Podstawowe wywołanie API | Korzyść |
|------------|--------------------------|---------|
| **Automatyczne generowanie raportów** | `convert(wordFile, pdfOptions)` | Tworzy szablony Word w raporty PDF w locie. |
| **Migracja danych ze starszych systemów** | `convert(excelFile, pdfOptions)` | Zachowuje arkusze kalkulacyjne jako niezmienialne PDF‑y dla zgodności. |
| **Archiwizacja e‑maili** | `convert(emailMessage, pdfOptions)` | Przechowuje e‑maile jako przeszukiwalne PDF‑y z opcjonalnym wyodrębnianiem metadanych. |
| **Zbieranie metadanych** | `extractMetadata(pdfFile)` | Uzupełnia systemy zarządzania dokumentami o autora, datę utworzenia i liczbę stron. |

## Szybki start

1. **Dodaj zależność Maven** (lub odpowiednik Gradle) dla GroupDocs.Conversion.  
2. **Utwórz instancję `Converter`** z kluczem licencyjnym.  
3. **Wybierz format źródłowy** (np. `.docx`, `.xlsx`, `.csv`).  
4. **Skonfiguruj `PdfConvertOptions`** – ustaw rozmiar strony, podstawianie czcionek lub obsługę haseł.  
5. **Wywołaj `convert`** i zapisz strumień wyjściowy do pliku lub odpowiedzi.

> **Wskazówka:** Gdy potrzebujesz **convert excel to pdf java** z własnymi czcionkami, ustaw `PdfConvertOptions.setFontSubstitution(true)` i podaj `FontMapper`, aby zapewnić spójność wizualną.
